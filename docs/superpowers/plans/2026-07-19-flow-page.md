# /flow Product Flow Viewer Implementation Plan

> **For agentic workers:** REQUIRED SUB-SKILL: Use superpowers:subagent-driven-development (recommended) or superpowers:executing-plans to implement this plan task-by-task. Steps use checkbox (`- [ ]`) syntax for tracking.

**Goal:** Add a `/flow` page that shows VisaVista's four core flows (Login, Add a Student, Case Pipeline, Document Checklist) as rows of real, live-rendered app screens connected by arrows — a Mobbin-style internal reference for demoing the app to a teammate.

**Architecture:** A static config (`flowsConfig.js`) lists each flow's ordered screens as `{ label, path, caption }`. `FlowPage.jsx` bootstraps a demo session (so embedded screens never show a login wall) and renders one `FlowSection` per flow; each `FlowSection` renders a horizontal row of `FlowScreenTile`s, each an `<iframe>` pointing at the app's own real routes, rendered at full desktop size and scaled down via CSS transform, with a click-blocking overlay. Two existing pages (`StudentsPage`, `CaseDetailPage`) gain optional query-param hooks so the iframes can land directly on the "modal open" / "Documents tab" states without needing simulated clicks.

**Tech Stack:** React 18, react-router-dom v6, Tailwind (existing token classes only), lucide-react icons. No test framework exists in this repo (`package.json` has no vitest/jest) — verification is `npm run build` (catches syntax/import errors) plus manual browser confirmation, consistent with how the rest of this MVP is verified.

## Global Constraints

- No `fetch` outside `src/services/apiClient.js` — this plan adds no new fetch call; the session bootstrap reuses the existing `authService`.
- No hardcoded hex colors in `src/features/**` / `src/routes/**` / `src/components/**` — use token-mapped Tailwind classes (`bg-canvas`, `text-ink-muted`, `border-hairline`, etc.), matching the rest of the codebase.
- Fixed structural dimensions (panel widths, thumbnail sizes) use Tailwind's arbitrary-value bracket syntax (e.g. `w-[320px]`), the same pattern already used in `src/features/ask/components/AskPanel.jsx:120` (`w-[380px]`) — this is the codebase's existing convention for one-off layout sizes, distinct from the "no hardcoded color/spacing values" rule.
- Square corners: `rounded-none` on any new bordered container.
- Feature-first folders: new feature-specific files go under `src/features/flow/`; the new route file goes in `src/routes/`.
- The four flows and their exact screen sequences are fixed by the spec (`docs/superpowers/specs/2026-07-19-flow-page-design.md`) — do not add, remove, or reorder flows/screens without checking back against that spec.

### Task 4: FlowPage route component

**Files:**
- Create: `src/routes/FlowPage.jsx`

**Interfaces:**
- Consumes: `authService` from `src/features/auth/authService.js` (existing — `authService.me(): Promise<User>`, `authService.login({ email?, role? }): Promise<User>`, both already used by `AuthContext.jsx`), `FLOWS` from Task 1, `FlowSection` from Task 3, `PageHeader` from `src/components/ui/PageHeader.jsx` (existing, props: `{ eyebrow, title, description, actions, className }`), `Spinner` from `src/components/ui/Spinner.jsx` (existing, props: `{ size = 20 }`).
- Produces: default export `FlowPage()` — consumed by `App.jsx` (Task 5) as the element for the `/flow` route.

- [ ] **Step 1: Create the page**

On mount, checks for an existing session via `authService.me()`; if that rejects (no session), logs in as Admin so the embedded iframes never hit a login wall. Shows a spinner until that check resolves, then renders one `FlowSection` per flow.

```jsx
// src/routes/FlowPage.jsx
import { useEffect, useState } from 'react';
import PageHeader from '../components/ui/PageHeader.jsx';
import Spinner from '../components/ui/Spinner.jsx';
import { authService } from '../features/auth/authService.js';
import { FLOWS } from '../features/flow/flowsConfig.js';
import FlowSection from '../features/flow/components/FlowSection.jsx';

export default function FlowPage() {
  const [ready, setReady] = useState(false);

  useEffect(() => {
    let alive = true;
    authService.me()
      .catch(() => authService.login({ role: 'admin' }))
      .finally(() => { if (alive) setReady(true); });
    return () => { alive = false; };
  }, []);

  if (!ready) {
    return (
      <div className="flex h-screen items-center justify-center bg-canvas">
        <Spinner />
      </div>
    );
  }

  return (
    <div className="min-h-screen bg-canvas">
      <PageHeader
        eyebrow="Internal"
        title="VisaVista — Product Flows"
        description="How the app actually works, screen by screen."
      />
      <div className="flex flex-col gap-xl p-xl">
        {FLOWS.map((flow, i) => (
          <FlowSection key={flow.id} index={i + 1} flow={flow} />
        ))}
      </div>
    </div>
  );
}
```
